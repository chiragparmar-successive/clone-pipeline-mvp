ROLE:
You are a design-system normalization engine.

INPUT:
{{COMPONENTS}}

TASK:
Consolidate page-level component detections into a global, reusable component registry and page layouts.

NORMALIZATION RULES:
1) Merge synonyms:
   - header/nav/top-nav -> navbar
   - cards/features/grid -> card-grid
   - call-to-action/banner -> cta-banner
2) Preserve semantic differences (faq != testimonial).
3) Union props for merged components, but keep prop names stable and generic.
4) Remove style or implementation props.
5) Generate per-page layout as an ordered list of normalized component types.

PROP CLEANUP RULES:
- keep: title, subtitle, description, image, imageAlt, ctaLabel, ctaUrl, items, question, answer
- drop: className, css, styles, bgColor, margin, padding, shadow, radius, animation

    }
  },

    {
      "url": "/",
      "layout": ["navbar", "hero", "feature-section", "cta-banner", "footer"]
    }
  ]
}

OUTPUT RULES:
- Return valid JSON only.
- No commentary.
- No markdown.

VALIDATION GATES:
- No duplicate component keys in `components`.
- Each `pages[].layout` item must exist in `components`.
